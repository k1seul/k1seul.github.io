<h2 id="review">Review</h2>
<h3 id="abstract">Abstract</h3>
<p>How do we use multiple models for better predictions?<br>
-&gt; The simplest way of doing this is to average every prediction of models in the model ensemble(cluster).<br>
-&gt; However, this method is computationally expensive. Thus, the end user(or local user) may have difficulty using these models.</p>
<h4 id="then-can-we-compress-this-ensemble-of-models-into-a-single-model-how-do-we-distill-the-knowledge-learned-by-these-ensembles-of-models-to-one-network">Then can we compress this ensemble of models into a single model? (How do we distill the knowledge learned by these ensembles of models to one network?)</h4>
<p><img src="https://miro.medium.com/v2/resize:fit:2000/1*T8NnkvKIRmZT38EZ9GDoZg.png" alt="Bagging vs. Boosting: The Power of Ensemble Methods in Machine Learning |  by Thomas A Dorfer | Towards AI">     <em>Model Ensemble</em><br>
Models consist of experts and specialists (fine-grain classification or feature selection) which can trained in parallel.</p>
<h3 id="distillation-as-transfer-of-knowledge">Distillation as transfer of knowledge</h3>
<p>A large ensemble of models(cumbersome models)<br>
-&gt; can be trained separately with very strong regularizers such as dropout<br>
small model<br>
-&gt; trained through knowledge transfer from a large ensemble of models (distillation)</p>
<h3 id="concept">Concept</h3>
<p>how do we transfer knowledge? what is knowledge?</p>
<h4 id="knowlege-transfer-from-dataset-to-network-parameters">Knowlege transfer from dataset to network parameters</h4>
<p>Dataset(knowledge) -&gt; model ensemble (cumbersome model)</p>
<p>from the data view, knowledge is a function mapping input vectors to output vectors.<br>
-&gt; How to learn (train parameters) to this knowledge?<br>
-&gt; We use an objective function</p>
<p>But what if all the knowledge we need is not in the datasets we have?<br>
-&gt; Models should be learned to generalize well to new data. However, this requires more information than the objective functions we currently use.<br>
-&gt; But if we use an ensemble of models, we might have clues about ways to generalize well.<br>
<img src="https://pass.01consulting.co.kr/2018/guide/images/z_img01.gif" alt="김영일교육컨설팅(www.01consulting.co.kr - e대학합격컨설팅)"><br>
<em>학생성적분포</em><br>
-&gt; If we have only one student to learn the material, the student might not learn well. But if we use an ensemble of students, we might get clues about how to understand the subject well.<br>
<img src="https://velog.velcdn.com/images%2Fahp2025%2Fpost%2F872cc5ea-f5c7-4b8f-8b54-65ec4a082859%2Fknowledge_distillation.png" alt="Distilling the Knowledge in a Neural Network 논문 리뷰"><br>
<em>model distillation</em><br>
-&gt; If soft targets have high entropy(high variance), it is likely to have more information about training data<br>
-&gt; Then small model can be learned with high learning rate</p>
<p>How to use these small difference (high entropy) data to train small models? (transfer information)</p>
<ul>
<li>Problems with final softmax function -&gt; low values of input of softmax functions lead to output becoming 0<br>
this problem is solved by using final logit(value before input to the softmax) to make transfer dataset (optimization is done by minimizing squared error between two logit values)</li>
</ul>
