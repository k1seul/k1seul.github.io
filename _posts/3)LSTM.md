<h2 id="review">Review</h2>
<h3 id="abstract">Abstract</h3>
<h4 id="problem-">Problem :</h4>
<p>Dealing with sequential time dependent data<br>
-&gt; when updating using back-propagation, updates takes very long time. (decaying error back flow, gradient vanishes when time moves backward)</p>
<h3 id="solution-proposed">Solution Proposed</h3>
<p>Long Short-Term Memory (LSTM) model<br>
-&gt; Can learn to more than 1000 discrete time-step using constant error flow.<br>
-&gt; Computationally light due to it being <span class="katex--inline"><span class="katex"><span class="katex-mathml"><math xmlns="http://www.w3.org/1998/Math/MathML"><semantics><mrow><mi>O</mi><mo stretchy="false">(</mo><mn>1</mn><mo stretchy="false">)</mo></mrow><annotation encoding="application/x-tex">O(1)</annotation></semantics></math></span><span class="katex-html" aria-hidden="true"><span class="base"><span class="strut" style="height: 1em; vertical-align: -0.25em;"></span><span class="mord mathnormal" style="margin-right: 0.02778em;">O</span><span class="mopen">(</span><span class="mord">1</span><span class="mclose">)</span></span></span></span></span> per time step<br>
-&gt;   Way better then other RNN algorithms</p>
<h2 id="introduction">Introduction</h2>
<p>Recurrent Neural Network (Vanilla)<br>
-&gt; can use to store representations of recent inputs (short-term memory)<br>
-&gt; How about long term representation? X</p>
<h3 id="detailed-problem">Detailed Problem</h3>
<p>Vanilla RNN has one of following two problems<br>
error signals tend to</p>
<ol>
<li>blow up (oscillating weights since update happens dramatically)</li>
<li>vanish (cannot learning anything)</li>
</ol>
